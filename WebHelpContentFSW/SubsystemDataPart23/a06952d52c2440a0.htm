Single Subframe Mode

Turns the evaluation of a single subframe only on and off.

Evaluating a single subframe only improves the measurement speed. For successful synchronization, the subframe must be located within the captured data (= 1.2 ms). You can make sure that this is the case by using, for example, an external frame trigger signal.

For maximum measurement speed, the application turns off Auto According to Standard and sets the Number of Frames to Analyze to 1. These settings prevent the application from capturing data more than once for a single run measurement.

Remote command: 

[SENSe:​][LTE:​]FRAMe:​SSUBframe
