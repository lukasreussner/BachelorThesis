Trigger Offset

Defines the time offset between the trigger event and the start of the measurement.

Note: In MSRA/MSRT mode, the trigger offset defines the start of the measurement only for the MSRA/MSRT Master. For MSRA/MSRT applications, use the "Capture Offset" instead.

Note: When using an external trigger, the trigger offset is particularly important to detect the frame start correctly! (See "Trigger settings".) The R&S FSW GSM application expects the trigger event to be the start of the "active part" in slot 0.

Offset > 0:

Start of the measurement is delayed

Offset < 0:

Measurement starts earlier (pretrigger)

Remote command: 

TRIGger[:​SEQuence]:​HOLDoff[:​TIME]
