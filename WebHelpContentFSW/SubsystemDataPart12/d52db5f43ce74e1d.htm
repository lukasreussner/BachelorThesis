Question: How can I demodulate an FSK modulated signal using the "Manchester" code?

The R&S FSW VSA application does not provide a specific configuration setting to support this code. However, if you configure FSK modulation and use twice the usual symbol rate, the signal is demodulated correctly. Remember to consider the double symbol rate when evaluating the results, however! Only the "Symbol Table" and the bitstream results returned by the TRACe<n>[:​DATA] command are not interpreted correctly and must be demodulated manually (for example using a script).
