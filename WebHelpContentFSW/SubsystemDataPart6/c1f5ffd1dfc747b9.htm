Linked Markers

If enabled, the markers in all diagrams are linked, i.e. when you move a marker in one window, the markers in all other windows are moved to the same x-value.

Note: Since the group delay display uses a frequency grid with an offset of (<carrier_spacing>/2), the x-value of a linked marker on the group delay trace is always half the carrier spacing higher than in other result displays.

Linking markers is only possible if compatible reference data is available, or no reference data at all. Compatible reference data means the settings used to perform calibration were the same as the currently active measurement settings. If measurement settings are changed that do not match the reference data, linking markers is no longer possible and the function is automatically deactivated.

An exception to this rule is the center frequency. If a different center frequency was used for calibration, the markers can still be linked. However, in this case the markers are not moved to a specific x-value, but to the same horizontal position relative to the diagram edge.

Remote command: 

CALCulate<n>:​MARKer<m>:​LINK
