Using Markers

Note that the suffix at CALCulate has an effect only if you query the characteristics of a marker. If you set a marker, you can ignore the suffix because the markers are linked to each other over all measurement windows and will always be on the same frequency.

List of Commands
CALCulate<n>:MARKer<m>:AOFF
CALCulate<n>:MARKer<m>[:STATe]
CALCulate<n>:MARKer<m>:TRACe
CALCulate<n>:MARKer<m>:X
CALCulate<n>:MARKer<m>:Y?
