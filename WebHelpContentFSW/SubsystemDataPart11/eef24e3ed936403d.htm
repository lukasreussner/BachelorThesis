CALCulate<n>:CHRDetection:TABLe:RESults? [<Start>, <End>]

This command queries the chirp results table. The result is a comma-separated list of value sets, one set for each chirp.

If no query parameters are specified, the results for all detected chirps are returned.

Which values are returned depends on the enabled parameters for the results tables (see CALCulate<n>:​CHRDetection:​TABLe:​COLumn).

Suffix: 
<n>
irrelevant
Query parameters: 
<Start>
integer

The chirp number of the first chirp to be returned. Chirp numbers start at 1.

<End>
integer

The chirp number of the last chirp to be returned.

Return values: 
<1_ID>
<char_data>

Timestamp which corresponds to the absolute time the beginning of the chirp was detected

<2_ChirpNo>

Consecutive number of detected chirp, starts at 1 for each new measurement

<3_StateIndex>

Consecutive number of corresponding nominal chirp state as defined in the "Chirp States" table (see CALCulate<n>:​CHRDetection:​STATes[:​DATA])

<4_Begin>
<char_data>

Time offset from the analysis region start at which the signal first enters the tolerance area of a nominal chirp

Default unit: ms
<5_Length>
<char_data>

The duration of a chirp from begin to end, that is, the time the signal remains in the tolerance area of a nominal chirp.

Default unit: ms
<6_CRate>
<char_data>

Derivative of the FM vs time trace within the frequency measurement range

Default unit: kHz/μs
<7_CRateDev>

Deviation of the detected chirp rate from the nominal chirp state (in kHz/us).

For details see "Chirp State Deviation".

Default unit: kHz/μs
<8_FreqAvg>

Average frequency measured within the frequency measurement range of the chirp

Default unit: kHz
<9_BW>

Bandwidth occupied by the chirp.

Default unit: kHz
<10_FMSetPoint>

The FM settling point is the point where FM deviation does not exceed the FM settling tolerance anymore.

Default unit: ms
<11_FMSetTime>

The FM settling time refers to the time interval between the detected chirp start and the FM settling point.

Default unit: ms
<12_FMSetLeng>

The FM settled length is determined as the time interval starting from the FM settling point until the point where the deviation exceeds the settling tolerance (chirp end).

Default unit: ms
<13_FMDevMax>

Maximum deviation of the chirp frequency from the nominal chirp frequency as defined in the "Chirp States" table. The deviation is calculated within the frequency measurement range of the chirp.

For details see "Frequency Deviation (Peak)".

Default unit: kHz
<14_FMDevRMS>

RMS deviation of the chirp frequency from the nominal (linear) chirp frequency as defined in the "Chirp States" table. The deviation is calculated within the frequency measurement range of the chirp.

For details see "Frequency Deviation (RMS)".

Default unit: kHz
<15_FMDevAvg>

Average deviation of the chirp frequency from the nominal (linear) chirp frequency as defined in the "Chirp States" table. The deviation is calculated within the frequency measurement range of the chirp.

For details see "Frequency Deviation (Average)".

Default unit: kHz
<16_FreqINLMax>

The maximum frequency integrated non-linearity (INL) indicates the maximum deviation of the measured chirp from the nominal chirp, in relation to the chirp bandwidth.

Default unit: none
<17_FreqINLRMS>

The RMS frequency integrated non-linearity (INL) indicates the RMS deviation of the measured chirp from the nominal chirp, in relation to the chirp bandwidth.

Default unit: none
<18_FreqINLAvg>

The average frequency integrated non-linearity (INL) indicates the average deviation of the measured chirp from the nominal chirp, in relation to the chirp bandwidth.

Default unit: none
<19_PMSetPoint>

The PM settling point is the point where FM deviation does not exceed the PM settling tolerance anymore.

Default unit: ms
<20_PMSetTime>

The PM settling time refers to the time interval between the detected chirp start and the PM settling point.

Default unit: ms
<21_PMSetLeng>

The PM settled length is determined as the time interval starting from the PM settling point until the point where the deviation exceeds the settling tolerance (chirp end).

Default unit: ms
<22_PMDevMax>

Maximum deviation of the chirp phase from the nominal chirp phase as defined in the "Chirp States" table. The deviation is calculated within the frequency measurement range of the chirp.

For details see "Phase Deviation (Peak)".

Default unit: kHz
<23_PMDevRMS>

RMS deviation of the chirp phase from the nominal (linear) chirp phase as defined in the "Chirp States" table. The deviation is calculated within the frequency measurement range of the chirp.

For details see "Phase Deviation (RMS)".

Default unit: kHz
<24_PMDevAvg>

Average deviation of the chirp phase from the nominal (linear) chirp phase as defined in the "Chirp States" table. The deviation is calculated within the frequency measurement range of the chirp.

For details see "Phase Deviation (Average)".

Default unit: kHz
<25_PowMin>

Minimum power level measured during a chirp. Which part of the chirp precisely is used for calculation depends on the power parameters in the "Power" measurement range configuration.

Default unit: dBm
<26_PowMax>

Maximum power level measured during a chirp. Which part of the chirp precisely is used for calculation depends on the power parameters in the "Power" measurement range configuration.

Default unit: dBm
<27_PowAvg>

Average power level measured during a chirp. Which part of the chirp precisely is used for calculation depends on the power parameters in the "Power" measurement range configuration.

Default unit: dBm
<28_PowRip>

Power level measured during the chirp ripple time. Which part of the chirp precisely is used for calculation depends on the power parameters in the "Power" measurement range configuration.

Default unit: dBm
