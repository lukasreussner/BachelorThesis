Average Mode

Defines the mode with which the trace is averaged over several sweeps.

This setting is generally applicable if trace mode "Average" is selected. For FFT sweeps, the setting also affects the VBW (regardless of whether or not the trace is averaged).

(See also Video Bandwidth (VBW)).

How many sweeps are averaged is defined by the "Sweep/Average Count".

For details see How Trace Data is Averaged - the Averaging Mode.

"Linear"

The power level values are converted into linear units prior to averaging. After the averaging, the data is converted back into its original unit.

"Logarithmic"

For logarithmic scaling, the values are averaged in dBm. For linear scaling, the behavior is the same as with linear averaging.

"Power"

Activates linear power averaging.

The power level values are converted into unit Watt prior to averaging. After the averaging, the data is converted back into its original unit.

Use this mode to average power values in Volts or Amperes correctly.

In particular, for small VBW values (smaller than the RBW), use power averaging mode for correct power measurements in FFT sweep mode.

Remote command: 

[SENSe:​]AVERage<n>:​TYPE
