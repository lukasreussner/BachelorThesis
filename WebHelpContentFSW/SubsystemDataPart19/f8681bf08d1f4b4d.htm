CALCulate<n>:LIMit<li>:FAIL?

This command queries the result of a limit check in the specified window.

To get a valid result, you have to perform a complete measurement with synchronization to the end of the measurement before reading out the result. This is only possible for single measurement mode.

See also INITiate<n>:​CONTinuous.

Suffix: 
<n>
1..n

Window

<li>
1..n

SEM: Limit line according to Table "Limit line suffix <k> for SEM measurements in WLAN application"

Limit line for unused tone error RU group:

// 1 = -35dB LHS

// 2 = RUIdx-3

// 3 = RUIdx-2

// 4 = RUIdx-1

// 5 = RUIdx

// 6 = RUIdx+1

// 7 = RUIdx+2

// 8 = RUIdx+3

// 9 = -35dB RHS

Return values: 
<Result>
OFF | 0

PASS

ON | 1

FAIL

Example: INIT;*WAI

Starts a new sweep and waits for its end.

CALC2:LIM4:FAIL?

Queries the result of the check for spectrum flatness (upper) limit line in window 2.

Usage: Query only
Manual operation: See "Unused Tone Error Summary"
See "Spectrum Emission Mask"
